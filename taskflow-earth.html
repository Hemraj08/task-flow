Features

Everything you need to stay productive

A powerful REST API, beautiful UI, and smart tools to help you manage your day.

⚡

Lightning Fast CRUD

Full REST API with create, read, update, and delete operations. Instant responses powered by MongoDB.

🔐

JWT Authentication

Secure user authentication with JSON Web Tokens. Your data is always protected and private.

🏷️

Smart Prioritization

Tag tasks as High, Medium, or Low priority. Filter, sort, and focus on what matters most.

Categories & Labels

Organize work into custom categories. Keep personal and professional tasks cleanly separated.

REST API Access

REST API Reference

All endpoints follow REST conventions. Base URL: https://api.taskflow.app/v1

🔐
Authentication Required
Include your JWT token: Authorization: Bearer <token>

Authentication

POST /auth/register Register a new user 
// Request body
{
  "name": "Jane Doe",
  "email": "jane@example.com",
  "password": "securepassword"
}

// Response 201
{
  "success": true,
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "user": { "id": "64a7...", "name": "Jane Doe", "email": "jane@example.com" }
}
POST /auth/login Login and get JWT token 
// Request body
{
  "email": "jane@example.com",
  "password": "securepassword"
}

// Response 200
{
  "success": true,
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "user": { "id": "64a7...", "name": "Jane Doe" }
}

Tasks

GET /tasks Get all user tasks 
// Query params: ?priority=high&category=work&status=active

// Response 200
{
  "success": true,
  "count": 12,
  "tasks": [
    {
      "_id": "64a7f3...",
      "title": "Design landing page",
      "description": "Create wireframes and mockups",
      "priority": "high",
      "category": "work",
      "status": "active",
      "dueDate": "2024-12-31T00:00:00Z",
      "createdAt": "2024-11-01T08:30:00Z"
    }
  ]
}
POST /tasks Create a new task 
// Request body
{
  "title": "Review pull request",
  "description": "Check PR #47 for code quality",
  "priority": "medium",
  "category": "work",
  "dueDate": "2024-12-15"
}

// Response 201
{
  "success": true,
  "task": { "_id": "64a7...", "title": "Review pull request", ... }
}
GET /tasks/:id Get task by ID 
// Response 200
{
  "success": true,
  "task": {
    "_id": "64a7f3b2c1d8e9f0a1b2c3d4",
    "title": "Review pull request",
    "status": "active",
    "priority": "medium",
    ...
  }
}

// Response 404
{ "success": false, "error": "Task not found" }
PUT /tasks/:id Update a task 
// Request body (partial updates supported)
{
  "title": "Review PR #47 and leave comments",
  "status": "completed",
  "priority": "high"
}

// Response 200
{
  "success": true,
  "task": { "_id": "64a7...", "title": "Review PR #47...", "status": "completed" }
}
DELETE /tasks/:id Delete a task 
// Response 200
{ "success": true, "message": "Task deleted successfully" }

// Response 404
{ "success": false, "error": "Task not found" }
